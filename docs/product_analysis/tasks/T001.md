## рҹҗӣ PROBLEMAS ENCONTRADOS E SOLUГҮГ•ES

### Problema 1: Edge renderizando como HTML puro

**Erro:** O conteГәdo Edge aparecia como texto na tela ao invГ©s de ser processado.

**Causa:** O arquivo `resources/views/components/layout.edge` havia sido removido acidentalmente durante a limpeza de templates.

**SoluГ§ГЈo:** Recriar o arquivo `resources/views/components/layout.edge` com a sintaxe correta do Edge.js v7.

**LiГ§ГЈo:** No AdonisJS v7, `@layout()` resolve automaticamente para `resources/views/components/layout.edge`. A pasta `layouts/` **nГЈo Г© usada** para este padrГЈo.

---

### Problema 2: `Cannot read properties of undefined (reading 'name')`

**Erro:** Ao acessar a pГЎgina de login sem estar autenticado.

**Causa:** 
1. O template usava `@loggedIn`/`@guest` que nГЈo funcionam como esperado no contexto de layout como componente
2. O campo `auth.user.name` nГЈo existe вҖ” o modelo User usa `fullName`

**SoluГ§ГЈo:**
```edge
@if(auth.user)
  <span>OlГЎ, {{ auth.user.fullName ?? auth.user.email.split('@')[0] }}</span>
@else
  <a href="{{ route('login') }}">Entrar</a>
@endif
```

**LiГ§ГЈo:** 
- Usar `@if(auth.user)` ao invГ©s de `@loggedIn`/`@guest` no layout
- Usar fallback com `??` para quando `fullName` Г© null
- O campo no modelo Г© `fullName`, nГЈo `name`

---

### Problema 3: `Cannot lookup route "login"`

**Erro:** Ao tentar gerar URLs para rotas de autenticaГ§ГЈo.

**Causa:** As rotas nГЈo tinham nomes definidos (faltava `.as('login')`).

**SoluГ§ГЈo:** Adicionar nomes a todas as rotas pГәblicas:
```ts
router.get('login', [controllers.Session, 'create']).as('login')
router.get('signup', [controllers.NewAccount, 'create']).as('signup')
router.post('logout', [controllers.Session, 'destroy']).as('logout')
```

**LiГ§ГЈo:** Sempre nomear rotas que serГЈo referenciadas por `route('nome')` nos templates Edge.

---

### Problema 4: `@section('head')` renderizando como texto

**Erro:** O texto `@section('head') @end` aparecia na tela como conteГәdo HTML.

**Causa:** No Edge.js v7 com layouts como componentes, as `@section`s podem nГЈo funcionar como esperado quando usadas diretamente no layout principal.

**SoluГ§ГЈo:** Remover `@section('head')` e `@section('scripts')` do layout. Usar `{{ title ?? 'Muvia' }}` diretamente no `<title>`.

**LiГ§ГЈo:** Evitar `@section` no layout principal quando usando `@layout()` como componente. Preferir interpolaГ§ГЈo direta ou props.

---

### Problema 5: TypeScript errors nos controllers

**Erro:** `response.redirect().toRoute()` nГЈo existe no AdonisJS v7.

**Causa:** API de redirect mudou no v7.

**SoluГ§ГЈo:**
```ts
// Antes (incorreto)
response.redirect().toRoute('home')

// Depois (correto)
response.redirect('/')
```

**LiГ§ГЈo:** Verificar a API correta do AdonisJS v7 antes de usar mГ©todos de redirect.

---

### Problema 6: TypeScript error no logger

**Erro:** `Type 'URL' is not assignable to type 'string | number'` ao usar `new URL()` no `destination`.

**Causa:** A API `targets.file()` espera string ou number, nГЈo URL object.

**SoluГ§ГЈo:**
```ts
// Antes (incorreto)
targets.file({
  destination: new URL('../logs/app.log', import.meta.url),
  level: 'info',
})

// Depois (correto)
targets.file({
  destination: app.makePath('logs/app.log'),
})
```

**LiГ§ГЈo:** Usar `app.makePath()` ao invГ©s de `new URL()` para caminhos de arquivos no AdonisJS v7.

---

### Problema 7: Pastas duplicadas na raiz do projeto

**Erro:** Diversas pastas vazias e desnecessГЎrias na raiz do projeto:
- `appadapters/`, `appdtos/`, `appevents/`, `appjobs/`, `applisteners/`
- `appservicesauth/`, `appservicesdemandas/`, `appserviceskyc/`, etc.
- `databaseseeders/`, `publicassets/`, `publicimages/`
- `resourcesviewscomponents/`, `resourcesviewsemails/`, `resourcesviewslayouts/`
- `resourcesviewspagesauth/`, `resourcesviewspagesdemandas/`, etc.
- `tmpuploads/` e diversos arquivos `.gitkeep` correspondentes

**Causa:** O projeto foi inicializado com pastas duplicadas na raiz que nГЈo refletem a estrutura real do AdonisJS. Por exemplo, `appadapters/` na raiz ao invГ©s de usar `app/adapters/`.

**LiГ§ГЈo:** Sempre usar a estrutura padrГЈo do AdonisJS (`app/adapters/`, `app/dtos/`, etc.) e nГЈo criar pastas separadas na raiz do projeto.
